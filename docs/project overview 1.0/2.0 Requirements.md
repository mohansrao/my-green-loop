# 2.0 Requirements & Implementation Status

## Vision
Transform **My Green Loop** from a dining rental platform into a comprehensive **Sustainable Living Hub** that showcases eco-friendly products, curates educational content, builds community engagement, and tracks collective environmental impact.

---

## Implemented Features

### ✅ Feature 2: Content Hub & Resource Library
**Description**: A curated collection of sustainability-focused articles, videos, and educational resources from across the web. Users can discover, search, and bookmark content organized by topics.

**Delivered**:
- Pin external content (articles, videos) with rich metadata auto-fetched via URL scraping
- Multi-category organization with tagging system
- Search and filtering by topic, type, and sort order
- Featured/trending content sections
- User bookmarking via localStorage (no auth required)
- Admin interface for adding, editing, and categorizing content

---

### ✅ Feature 4: Environmental Impact Dashboard (`/impact`)
**Description**: Visualize collective environmental impact through live metrics.

**Delivered**:
- Real-time counters: waste diverted, CO₂ prevented, water saved
- Dynamic utilization bar based on actual rental data
- Impact stat equivalency comparisons (e.g., stack of plates vs. landmarks)
- Data sourced live from `rentals` + `rental_items` + `products` tables

---

### ✅ Admin Dashboard Enhancements
**Delivered across multiple iterations**:
- **Orders**: Full order history (no row limit), sortable/searchable, inline status dropdown (pending → confirmed → completed → cancelled), order detail modal, CSV export. All status mutations are server-side authenticated.
- **Inventory**: Stock monitoring with low-stock alerts.
- **Feedback Management**: Approve/hide customer submissions, analytics dashboard.
- **Categories**: Manage Content Hub taxonomy.
- **Settings** (`/admin/settings`): Live configuration of admin phone number, admin email, SMS notification toggle, max rental days, and max sets — no code or env var changes needed. Values stored in `app_settings` DB table.

---

### ✅ Catalog & Booking UX Improvements
**Delivered**:
- Date dropdowns replace calendar picker for better mobile usability
- End date filtered to prevent picking before start date
- Past dates blocked from start date picker
- 5-day rental cap enforced in UI dropdowns
- Customer's selected dates remembered if they navigate back from checkout
- Second "Complete Reservation" CTA added at top of checkout form

---

### ✅ Home Page Redesign
**Delivered**:
- "Why It Matters" section with mission statement and 3 eco impact points
- "How It Works" moved above the stat block for better flow
- Stat block: events served, sets available, service area
- "Community Impact" section: 6 most-recent confirmed rentals shown (anonymised) to build social proof
- All sections wired to live data via `/api/rentals/recent-impact`

---

### ✅ Notification System
**Delivered**:
- **SMS** (Twilio A2P 10DLC): Admin + customer notifications on order placement. Admin number configurable via Settings UI. Toggle on/off via Settings UI.
- **Email** (Nodemailer): Admin + customer emails on order placement. Multi-provider support (Gmail, Yahoo, Hotmail, Outlook) via `EMAIL_SERVICE` env var. Admin email configurable via Settings UI.
